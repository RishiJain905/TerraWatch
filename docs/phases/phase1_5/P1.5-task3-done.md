## What Was Implemented

### 1) Shared dedup module
Created `backend/app/core/dedup.py` with:
- `filter_stale_planes_open_sky()`
- `filter_stale_planes_adsblol()`
- `filter_stale_ships_digitraffic()`
- `filter_stale_ships_ais_friends()`
- `deduplicate_planes()`
- `deduplicate_ships()`

Behavior implemented:
- stale filtering before merge
- plane dedup by case-insensitive ICAO / `id`
- ship dedup by MMSI / `id`
- newer timestamp wins
- source priority on ties:
  - OpenSky for planes
  - Digitraffic for ships
- safe ship metadata backfill only when the winner is actually missing metadata
- malformed upstream records are skipped instead of crashing the whole batch
- future timestamps beyond the allowed skew are filtered out
- alias timestamp fallback support:
  - `time_position` for planes
  - `last_position` for ships
- dedup output includes `sources` provenance for audit/debug in the in-memory merged result

### 2) Dedicated Task 3 unit tests
Created `backend/tests/test_dedup.py`.

Coverage includes:
- all requested plane dedup scenarios
- all requested ship dedup scenarios
- stale filtering behavior
- same-source duplicate resolution
- malformed-record resilience
- source provenance on merged duplicates
- alias timestamp handling
- partial-source/missing-timestamp edge cases

### 3) Scheduler integration
Updated `backend/app/tasks/schedulers.py`.

Plane scheduler changes:
- `refresh_planes_once()` now fetches OpenSky and ADSB.lol in parallel
- merges them through `deduplicate_planes()`
- falls back to the surviving plane source if one upstream fetch fails
- now serializes plane DB writes with `db_write_guard()` like the ship path

Ship scheduler changes:
- removed the old in-file ship dedup logic from scheduler behavior
- `refresh_ships_once()` and `ingest_aisstream_batch()` now use the shared dedup module
- cached ship snapshots are stale-pruned through the shared filters before merge
- added retry behavior for the aisstream listener loop after unexpected listen failures
- improved scheduler startup so missing background tasks can be restarted without duplicating healthy ones
- improved cache safety so failed ship persistence does not advance in-memory snapshot state
- preserved existing persistence and batch-broadcast behavior

### 4) Scheduler test expansion
Updated `backend/tests/test_database_and_scheduler.py`.

Added/updated coverage for:
- plane dual-source merge behavior
- stale plane source filtering before persistence/broadcast
- one-plane-source failure fallback
- ship merge path using shared dedup logic
- aisstream batch ingestion using shared dedup logic
- ship cache rollback when persist fails
- aisstream listener retry after listen failure
- restarting a missing aisstream scheduler task without duplicating the others
- retaining the prior Digitraffic snapshot when a refresh returns empty
