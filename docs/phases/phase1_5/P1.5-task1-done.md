## What Was Implemented

### New ADSB.lol service
Created `backend/app/services/adsblol_service.py` with:
- `normalize_hex(hex_str: str) -> str`
- `AdsblolService` class
- async `fetch_planes()` method
- module-level `fetch_planes()` wrapper for repo consistency

### ADSB.lol normalization behavior
The service normalizes ADSB.lol aircraft payloads into the repo's existing `Plane` contract:
- `id`
- `callsign`
- `lat`
- `lon`
- `alt`
- `heading`
- `speed`
- `squawk`
- `timestamp`

Primary spec field handling:
- `hex` -> normalized for matching, stored as lowercase `id` to preserve current repo behavior
- `flight` -> `callsign`
- `lat` -> `lat`
- `lng` -> `lon`
- `alt` -> `alt`
- `dir` -> `heading`
- `speed` -> `speed`
- `squawk` -> `squawk`

Defensive fallback handling also supports:
- `lon`
- `alt_baro`
- `alt_geom`
- `track`
- `gs`

### Timestamp handling
Timestamp resolution order:
1. top-level `last_timestamp`
2. top-level `ctime`
3. per-record `last_timestamp`
4. per-record `ctime`
5. `utc_now_iso()` fallback

### Error handling
The service logs and returns `[]` on:
- request timeout
- HTTP error
- invalid JSON / parse failure
- unexpected top-level payload shape
- missing or invalid aircraft list
