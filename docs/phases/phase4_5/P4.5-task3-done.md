## Architecture Decision: Callback Pattern

Rather than lifting all 4 hooks from Globe.jsx to App.jsx (which would require passing 10+ WS callbacks back down), Globe.jsx exposes its filter controls via an `onFilterHooksReady` callback prop. This useEffect fires whenever any filter state changes:

```javascript
useEffect(() => {
  if (onFilterHooksReady) {
    onFilterHooksReady({
      planes:    { filters: planesFilters,    updateFilter: planesUpdateFilter },
      ships:     { filters: shipsFilters,     updateFilter: shipsUpdateFilter },
      events:    { filters: eventsFilters,    updateFilter: eventsUpdateFilter },
      conflicts: { filters: conflictsFilters, updateFilter: conflictsUpdateFilter },
    })
  }
}, [onFilterHooksReady, planesFilters, shipsFilters, eventsFilters, conflictsFilters])
```

App.jsx stores this in state and passes `filterHooks` to Sidebar. This keeps the WebSocket handler in Globe.jsx (where it belongs) while giving Sidebar full access to filter controls.

## Filter Controls Implemented

### Conflicts Panel (ConflictsFilterPanel)
- **Min Intensity**: Range slider (0-11, step 1) matching |tone|+1 weight formula

## Key Design Decisions

1. **Multiple panels open simultaneously** — `expandedPanels` state is an object `{ planes: bool, ships: bool, ... }`, not a single key. Users can open multiple filter panels at once.

2. **16 event categories, not 8** — The spec example code showed 8 simplified categories, but the actual useEvents hook uses 16 real categories from the GDELT research. All 16 are included.

3. **Spec typo fixed** — The spec had `</parameter>` instead of `</option>` in the ConflictsFilterPanel. Fixed.

4. **Null-safe rendering** — Filter panels only render when `filterHooks` is not null (guards against initial render before Globe mounts).

5. **Toggle vs expand separation** — Clicking the toggle switch only toggles the layer on/off. Clicking the chevron expands/collapses the filter panel. These are independent actions.
