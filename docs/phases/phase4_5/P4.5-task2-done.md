## Filter Specifications Implemented

### usePlanes
- `altitudeMin` (0) / `altitudeMax` (50000) — range filter on `plane.alt ?? 0`
- `callsign` ('') — case-insensitive partial match, `.trim()` applied
- `speedMin` (0) — minimum speed filter on `plane.speed ?? 0`

### useShips
- `types` (['cargo','tanker','passenger','fishing','other']) — array of enabled types; empty `ship_type` treated as 'other'
- `speedMin` (0) — minimum speed filter on `ship.speed`

### useEvents
- `categories` (all 16 GDELT categories) — array of enabled categories; events with empty category excluded
- `toneMin` (-10) / `toneMax` (10) — range filter on `event.tone || 0`
- `dateRange` ('all') — dropdown: '24h', '48h', '7d', 'all'; compares parsed event.date (YYYY-MM-DD) against cutoff

### useConflicts
- `intensityMin` (0) — filter by `Math.abs(conflict.tone || 0) + 1 >= intensityMin`
- `dateRange` ('all') — same date logic as events

## Key Design Decisions

1. **useEvents and useConflicts created as standalone hooks** — Previously events/conflicts were managed inline in Globe.jsx with useState. Research (Task 1) confirmed no dedicated hooks existed. Created them following the useShips.js pattern.

2. **Consistent updateFilter(key, value) API** — All 4 hooks expose `updateFilter(key, value)` for single-filter updates. Sidebar controls (Task 3) will call this.

3. **Raw data always preserved** — Each hook returns both raw (planes, ships, events, conflicts) and filtered (filteredPlanes, filteredShips, filteredEvents, filteredConflicts) arrays. Globe.jsx uses filtered data for layers, raw counts for the info bar.

4. **useMemo for performance** — All filtered arrays are memoized with `[rawData, filters]` dependencies to avoid re-computation on unrelated renders.

5. **GDELT date parsing** — Events use `YYYY-MM-DD` format strings, parsed via `new Date(event.date).getTime()` for cutoff comparison.
