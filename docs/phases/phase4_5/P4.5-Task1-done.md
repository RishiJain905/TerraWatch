## Key Findings

### Planes
- All 8 fields filterable: `alt` (feet), `callsign` (trim spaces), `speed` (knots), `lat`, `lon`, `heading`, `squawk`, `id`
- Altitude bands: <10k ft (green), 10–30k ft (yellow), >30k ft (red)
- `alt` defaults to 0 when null in OpenSky data
- Speed in knots at frontend (converted from m/s × 1.94384)

### Ships
- Filterable: `ship_type` (cargo/tanker/passenger/fishing/other), `speed` (knots)
- All AIS codes map to 5 frontend categories; `tug` and `sailing` fall through to `other`
- Speed is already in knots — no conversion needed

### Events (GDELT)
- Filterable: `category` (16 categories from GDELT code mapping), `tone` (float, approx −10 to +10), `date` (YYYY-MM-DD)
- Tone rendered as color: red (negative) → yellow-green (neutral) → green (positive)
- `mass_gvc` is actually TWO categories: `unconventional_mass_gvc` (code 10) and `conventional_mass_gvc` (code 12)

### Conflicts (GDELT Violent Subset)
- Same fields as Events + pre-filtered by backend to 6 violent categories
- **No `fatalities` field exists in GDELT data** — intensity uses `|tone| + 1` as weight
- Categories: assault, fight, unconventional_mass_gvc, conventional_mass_gvc, force_range, rioting

---

## Discrepancies vs. Phase 4.5 Overview

1. `fatalities` field does not exist in GDELT — conflict intensity is tone-driven only
2. `mass_gvc` = two separate categories (codes 10 and 12)
3. No `useEvents` or `useConflicts` hooks — events/conflicts managed in `Globe.jsx` with `useState`
4. Plane speed field is `speed`, not `gs`
5. GDELT event codes stored as 2-digit strings with leading zeros (e.g., `"08"`)
